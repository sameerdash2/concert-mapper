About this site

How does this website work?

The site looks up an artist name in the setlist.fm API and then retrieves all the setlists for that artist. Setlist.fm is a large community-maintained database of songs played at concerts, known as "setlists."

Why is the pin placement a little off?

While the data returned by setlist.fm includes names of venues, it only provides map coordinates that correspond to the city of the concert. This means pins will appear on the geographical center of the city, rather than the actual venue of the concert.

There is an edge case for when multiple concerts occur in the same city. To prevent their pins from overlapping, I "spread" the pins into a circle around the city's center. This strategy does sometimes push some pins into the ocean, but at least every pin can be clicked.

What tech does the site use?

The backend is a Python app (Flask) that queries the setlist.fm API. On the frontend, we use Leaflet (map embed), Pico CSS, and vanilla JS. The source code is available on GitHub.
